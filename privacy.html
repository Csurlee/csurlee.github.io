Privacy Policy

Last updated: April 10, 2026

WallWatch does not collect, transmit, or share any personal data with third parties. All communication happens directly between your device and your own pfSense firewall.

1. What WallWatch does

WallWatch is a mobile application that allows you to administer your own self-hosted pfSense firewall via the pfSense REST API v2. It connects exclusively to the firewall address you configure in the app settings.

2. Data we collect

We collect no data. Specifically:

  • No analytics or usage tracking
  • No crash reporting sent to any external server
  • No advertising identifiers
  • No location data
  • No contact information
  • No device identifiers stored or transmitted

3. Data stored on your device

WallWatch stores the following information locally on your device only:

  • Firewall address & port — stored in device AsyncStorage
  • API key or password — stored in the device's secure encrypted keystore (Android Keystore / iOS Secure Enclave) via Expo SecureStore. Never written to disk in plain text.
  • App preferences — refresh interval, theme, debug mode — stored in device AsyncStorage

This data never leaves your device except as authentication headers sent directly to your own firewall.

4. Network communication

The app communicates exclusively with the pfSense firewall IP address or hostname you provide. No requests are made to any WallWatch server, cloud service, or third party. All API calls go directly from your device to your firewall over your local network or VPN.

5. Third-party services

WallWatch is built with Expo and uses Expo's OTA update service to deliver app updates. When the app starts, it may check Expo's servers for a JS bundle update. No personal data is sent during this check — only the app's runtime version identifier.

6. Children's privacy

WallWatch is a professional network administration tool not directed at children under 13. We do not knowingly collect any information from children.

7. Changes to this policy

If this policy changes, the updated version will be published at this URL with a new "Last updated" date.
